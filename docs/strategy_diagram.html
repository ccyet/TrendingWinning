策略架构层级

从数据到交易分成五层：TDX 分钟 K 线负责真实行情，结构识别负责抽象形态，多周期决策负责过滤，执行层负责回测和参数扫描。

这部分用于定系统边界：每层只输出标准字段，下一层只消费字段，不直接耦合内部实现。
L0
数据源
TDX 分钟 K 线 5m / 15m / 30m / 60m，全程去掉 akshare 通道。
标准 OHLCV date、stock_code、open、high、low、close、volume、amount。
本地 parquet 缓存 按周期和复权方式落盘，扫描时批量读取。
L1
多周期状态
60m 定方向 判断主趋势是否允许做多，过滤逆势突破。
30m 找结构 识别通道、平台、压缩区和前高压力。
15m / 5m 找触发 突破、回踩确认、二次启动和止损位置。
L2
结构识别
标志K 放量、大振幅、实体强，后续可拆成涨停、反包、缺口、长阳。
趋势通道 当前为滚动回归通道，可替换为高低点趋势线或 Donchian 通道。
压力与保护位 上一根上轨、标志K低点、通道下轨、前高前低。
L3
决策触发
突破 trigger 收盘突破上一根上轨，同时量能满足阈值。
多周期共振 高周期不逆势，中周期有结构，低周期给入场。
交易过滤 是否回踩确认、是否允许高开、是否排除长上影。
L4
执行回测
入场与退出 触发 K 收盘入场，止损、止盈、最大持仓 K 数退出。
批量遍历 按股票池、周期、参数组合做高性能扫描。
Web 工作台 参数可视化、信号复盘、交易明细、后续 Docker 部署。
60m 只做趋势许可：上升或震荡偏上才允许做多。
30m 确认结构：通道抬升、压缩、平台突破前蓄势。
15m 识别标志K和突破前的成交量变化。
5m 执行 trigger：突破、回踩、止损位置更精细。

K线策略演示

这张图把策略落到 K 线上：先出现标志K，再形成上升通道，突破上一根通道上轨后触发入场，并用标志K低点或通道下轨做保护。

先按“60m 定方向、30m 画结构、15m 找标志K、5m 触发交易”画，具体口径你可以直接改右侧文字。
标志K口径 默认要求放量、大振幅、实体强；你可以改成“涨停板、反包、突破前高、低开高走”等更具体分类。
趋势通道口径 当前画法是上升通道。代码里先用滚动回归，后续可换成手工高低点通道、分形趋势线或箱体突破。
突破 trigger 当前是 close > prev_upper。可改为盘中触碰、收盘确认、次根确认、回踩上轨再买、二次突破再买。
止损与退出 当前先画成标志K低点或通道下轨保护位。可改成固定百分比、ATR、移动止损、跌破 5m 结构退出。

2. 标志K识别
放量 + 大振幅 + 实体
  • 振幅 ≥ 前 N 根均值 × range_multiple
  • 成交量 ≥ 前 N 根均值 × volume_multiple
  • 实体 / 全振幅 ≥ min_body_ratio
通过
当前定义的“标志K”

默认是强波动、强量能、实体足够大的 K。这里可能需要你重新定义：是否要求涨幅、是否允许长上影、是否区分反包/缺口/涨停。

输出字段
is_landmark
  • landmark_range_ratio
  • landmark_volume_ratio
  • landmark_body_ratio
3. 趋势通道
滚动回归中线

对最近 channel_lookback 根 close 做线性回归，得到通道中线和斜率。

扩展通道
上下轨
  • 残差包络：高低点偏离拟合线
  • ATR 包络：均值振幅 × band_atr_multiple
  • 取更宽的一侧，避免通道过窄
判方向
通道方向

slope > min_slope 为 up，slope < -min_slope 为 down，否则 flat。这里可改成结构高低点、N 字形、分形趋势线或人工通道。

4. 突破 Trigger
只看上一根通道上轨

突破判断使用 prev_channel_upper，避免当前突破 K 抬高通道后自我抵消。

同时满足
触发条件
  • close > prev_upper × (1 + buffer)
  • prev_direction ∈ up / flat
  • volume ≥ 前 N 根均量 × trigger_volume_multiple
  • 默认要求当前 K 同时是标志K
形成信号
breakout_trigger

当前只做向上突破。可补：失败突破、回踩确认、二次突破、下跌趋势反转、假突破过滤。

5. 回测与风控
入场

触发 K 的 close 作为 entry_price；同一标的持仓期间不重复开仓。

持仓管理
退出
  • 先检查 stop_loss
  • 再检查 take_profit
  • 最后 max_holding_bars 到期退出
输出
绩效

trade_count、win_rate、total_return、avg_return、max_drawdown。后续应补滑点、手续费、仓位、组合净值和批量参数扫描。

当前策略核心 先识别足够强的 K，再确认它突破上一根通道上轨。
当前弱点 标志K定义偏粗，趋势通道仍是线性回归口径。
优先修正点 标志K分类、趋势通道画法、突破后是否需要回踩确认。
工程边界 数据、信号、扫描、回测低耦合，参数可逐步扩展。

需要你定稿的策略口径

问题 当前实现 你可以修正成
标志K是否必须上涨？ 不要求，只看振幅、量能、实体。 可要求 close > open、涨幅阈值、突破前高、涨停/准涨停。
趋势通道如何画？ close 滚动线性回归 + 残差/ATR 包络。 可改为高低点通道、分形趋势线、Donchian、均线带。
突破是否直接买？ 突破 K 收盘买入。 可改为次根确认、回踩上轨、盘中触发、分批买入。
多周期如何共振？ 各周期独立扫描，只汇总最新状态。 可要求 60m 定方向、30m 找结构、15m/5m 触发。
退出规则是否合理？ 固定止盈、固定止损、固定持仓 K 数。 可改为通道下轨止损、标志K低点止损、移动止盈。
