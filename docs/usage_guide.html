TrendingWinning 使用指南

TDX-only A 股 K 线趋势策略工作台，覆盖数据落地、审计补齐、策略扫描、单策略回测、组合策略回测、参数遍历和 Docker 部署。

当前设计原则

路径选择使用文件夹选择框；同一任务的周期、代码、日期放在同一行；趋势、通道、区间、反转四类形态识别彼此解耦。

能力总览 本地运行 路径选择 TDX K线 策略扫描 单策略回测 组合策略回测 回测K线指南 输出产物 Docker Win/TDX 注意

能力总览

TDX 数据日 K 1d 和分钟 K 5m / 15m / 30m / 60m，不使用 akshare 通道；高分钟周期可由 TDX 5m 聚合。
数据管理本地缓存库存、parquet 审计、按需补齐、K 线覆盖率检查、最长连续缺口定位、日 K 锚定分钟交易日。
过滤规则按主板、科创、创业、北交所涨停开盘规则过滤污染交易日。
独立形态识别趋势、区间、通道、反转模块只输出标准事件，策略层统一消费。
挂单触发突破在信号 K 上方/下方挂单，撮合层按下一根 K 是否触发成交。
回测统计策略K线运行区间、横轴/价格轴缩放、大样本高清 K 线、开多/开空/止损/回撤止盈标注、交易方向过滤、逐笔交易、净值曲线、现金占比、订单拒绝和策略过滤。

本地运行

首次运行先安装依赖，再启动 Streamlit Web 应用。

cd /Users/a1234/Documents/TrendingWinning
python -m pip install -r requirements.txt
streamlit run streamlit_app.py --server.port 8520
Web 页面包含三个主栏目：TDX K线、策略扫描、回测。侧边栏只放全局数据目录、复权方式和 TDX 插件目录。Mac 本机不作为真实 TDX 取数环境，真实请求用 Parallels/Windows。

回测页的模块化字段和 K 线触发范例另见 backtest_kline_guide.html。

路径选择

所有文件夹路径都通过“选择文件夹”按钮弹出系统选择框，不再要求用户手写目录字符串。

位置用途交互方式
行情根目录本地 K 线仓库，例如 market/daily。点击“选择文件夹”弹出系统目录框；也可用快速位置、子文件夹和选定当前文件夹浏览。
TDX PYPlugins/userWindows 通达信 tqcenter 所在目录。Win 侧页面直接弹出本机目录框；Mac 侧 CLI 用 --runtime parallels 调度到 Win。
输出父目录保存回测或参数遍历产物。选择父目录，系统自动按实验名生成子文件夹。
如果当前会话没有可用桌面窗口或缺少 tkinter，页面会明确提示无法打开系统选择框，并继续提供站内目录浏览器。

TDX K线

先查看本地缓存库存，再用诊断确认 Windows/Parallels TDX 通道；随后做补齐计划，最后按需抓取缺失或质量异常的周期。

Mac 端 TDX 接口测试以 Parallels/Windows 通达信为准，先跑 tdx-doctor --runtime parallels，不要尝试从 Mac 本机通达信直接取数。若日 K 返回 ok、分钟 no_data，说明 Windows 通达信本地没有返回分钟 K 线；先确认 5m 分钟数据缓存，再做高周期请求或 5m 聚合。
python -m trending_winning.cli inventory-data \
  --symbols 000001.SZ,600519.SH \
  --timeframes 1d,5m,15m,30m,60m \
  --data-root /path/to/market/daily
python -m trending_winning.cli tdx-doctor \
  --symbols 000001.SZ,600519.SH \
  --timeframes 1d,5m,15m,30m,60m \
  --start "2026-05-25 09:30:00" \
  --end "2026-05-25 15:00:00" \
  --runtime parallels \
  --tdx-path "C:\\new_tdx64\\PYPlugins\\user"
python -m trending_winning.cli plan-data \
  --symbols 000001.SZ,600519.SH \
  --timeframes 1d,5m,15m,30m,60m \
  --start "2026-05-01 09:30:00" \
  --end "2026-05-25 15:00:00" \
  --min-coverage-ratio 0.95 \
  --data-root /path/to/market/daily
python -m trending_winning.cli prepare-data \
  --symbols 000001.SZ,600519.SH \
  --timeframes 1d,5m,15m,30m,60m \
  --start "2026-05-01 09:30:00" \
  --end "2026-05-25 15:00:00" \
  --min-coverage-ratio 0.95 \
  --data-root /path/to/market/daily
  • inventory-data 只列本地缓存库存；指定代码时缺失周期会显示 missing_file。
  • plan-data 只读取本地 parquet 和日 K 交易日锚点，不请求 TDX。
  • prepare-data 在 Mac 侧默认通过 Parallels 进入 Windows 执行真实 TDX 请求。
  • prepare-data 只请求缺失、质量错误或覆盖率低于门槛的标的周期。
  • 15m / 30m / 60m 原生周期无数据时，会按上午、下午交易段用 TDX 5m 聚合成目标周期。
  • 分钟周期会自动带上 1d 依赖，日 K 用于覆盖率锚点和涨停开盘过滤。
  • 所有入口会校验 start <= end，时间窗口写反会直接失败。

策略扫描

扫描页用于快速查看标志 K、趋势通道和突破触发。周期、标的、开始日期、结束日期在同一行，减少表单高度。

  • 周期：仅支持分钟级 5m / 15m / 30m / 60m。
  • 代码：支持逗号或换行分隔，例如 000001.SZ,600519.SH。
  • 结果：展示最新状态、触发数量和最近突破明细。

单策略回测

单策略只绑定一种形态识别模块，不进入组合仓位分配层。适合独立验证趋势、区间、通道或反转模块。

python -m trending_winning.cli single-backtest \
  --symbols 000001.SZ,600519.SH \
  --timeframe 30m \
  --higher-timeframe 60m \
  --higher-timeframe-max-age-minutes 120 \
  --start 2026-05-01 \
  --end 2026-05-25 \
  --detector trend \
  --risk-reward 2.0 \
  --max-holding-bars 12 \
  --max-actual-risk-pct 0.03 \
  --max-chase-pct 0.02 \
  --side-mode both \
  --min-coverage-ratio 0.95 \
  --fee-rate 0.0003 \
  --slippage-bps 5 \
  --output-dir runs/single-trend-001

推荐验证顺序

  1. 先关闭大周期方向过滤，只验证形态本身。
  2. 再开启 higher_timeframe，检查方向不一致和大周期信号过旧的过滤日志。
  3. 用 side_mode 在多/空、仅多、仅空之间切换，单独观察方向贡献。
  4. 最后加入真实风险、追价、手续费和滑点参数。

组合策略回测

组合回测把多个独立形态生成的订单统一进入组合层，组合层只负责排序、容量、资金、行业和重叠持仓约束。

python -m trending_winning.cli portfolio-backtest \
  --symbols 000001.SZ,600519.SH \
  --timeframe 30m \
  --start 2026-05-01 \
  --end 2026-05-25 \
  --detectors trend,range,channel,reversal \
  --risk-reward 2.0 \
  --max-open-positions 5 \
  --side-mode both \
  --capital-per-trade 0.25 \
  --risk-per-trade 0.01 \
  --reserve-cash 0.1 \
  --strategy-priority trend_signal_bar=1,range_signal_bar=2 \
  --strategy-capital-limit trend_signal_bar=0.6,range_signal_bar=0.4 \
  --sector-capital-limit 银行=0.5,新能源=0.4 \
  --symbol-sector-map 000001.SZ=银行,300750.SZ=新能源 \
  --output-dir runs/case-001 \
  --benchmark
  • capital_per_trade 是固定名义仓位。
  • risk_per_trade 按真实入场风险反推仓位。
  • allow_same_symbol_overlap 控制同一股票能否多策略重叠持仓。
  • side_mode 控制所有启用形态的交易方向：多/空、仅多或仅空。
  • 单策略和组合策略用信号 K 止损价与 risk_reward 计算目标价；固定百分比止盈止损只属于旧突破回测。
  • 回撤止盈由页面开关显式启用；关闭时启动浮盈和回撤幅度强制按 0 处理。启用后按上一根已完成 K 的峰值/谷值确认，当前 K 只刷新下一根可用的回撤线，避免同一根 K 同时启动和退出。
  • 单策略按满仓进出，统计 market_bar_count 和 exposure_bar_ratio，用于判断策略在样本时间轴内实际占用资金多久。
  • 组合净值按逐 K 盯市，统计现金占比、净暴露、真实总暴露、持仓数和资金贡献。

python -m trending_winning.cli replay-case \
  --case-configs runs/sweep-001/case_configs.jsonl \
  --case-config-hash <case_config_hash> \
  --output-dir runs/replay-001

Docker

docker run --rm -p 8520:8501 \
  -v /Users/a1234/Desktop/trend-backtest/data:/data \

容器内页面数据目录可选择 /data/market/daily。Docker 更适合读取已落地 parquet 后做扫描和回测，真实 TDX 取数仍走 Windows/Parallels。

Win/TDX 注意

  • 真实 TDX 请求依赖 Windows/Parallels 内通达信已启动并登录。
  • Mac CLI 默认 --runtime auto 会调度到 Parallels；需要本地强制运行时显式写 --runtime local。
  • 默认 Windows Python 是 C:\Users\Public\venvs\trending-winning\Scripts\python.exe，可用 --windows-python 覆盖。
  • 需要能定位 Windows PYPlugins/user，其中应能导入 tqcenter。
  • 在 Win 侧测试时，Web 文件夹选择器会浏览 Win 侧本机路径，不需要手写 macOS 路径。
  • 60m 会映射成 TDX 的 1h period。
  • 分钟线能否返回取决于客户端本地是否已有对应周期数据。
