趋势策略实战讲解：000852.SH

用默认趋势 detector 把“趋势背景、H1/H2 或 L1/L2 信号K、信号K上下方挂单、结构止损、2R 目标”连成一套可手工复核的交易流程。

样本说明

使用本地日 K：000852.SH，区间 2016-05-16 至 2026-05-22。指数日 K 缓存缺少成交量，本页为盘面教学图，正式回测仍按系统流动性过滤执行。

默认参数数学定义触发条件策略空间K线案例交易员检查表

默认参数

识别窗口trend_lookback = 20 根 K
趋势阈值trend_min_score = 1.0；高于阈值为多头趋势，低于 -1.0 为空头趋势
信号K质量strong_close_pos = 0.65；min_body_ratio = 0.45
回撤窗口pullback_lookback = 5；H2/L2 至少 2 段回撤或反弹
挂单方式多头：信号K高点 + 0.01；空头：信号K低点 - 0.01
风控结构止损来自 5 根 K 的回撤低点/反弹高点；risk_reward = 2.0；max_holding_bars = 12

数学定义转成盘面动作

趋势评分TrendScore = slope_z + structure_score + ma_alignment + close_strength + follow_through
多头信号TrendScore >= 1.0 AND close_pos >= 0.65 AND body_ratio >= 0.45 AND close > open
空头信号TrendScore <= -1.0 AND close_pos <= 0.35 AND body_ratio >= 0.45 AND close < open

开仓量化规则

开仓由四个条件合并判断

合并卡片把背景、信号K、挂单和风控放在同一视图里，交易员可以逐项核对，不需要在多个段落里来回找规则。

trend_ok_long = TrendScore >= trend_min_score trend_ok_short = TrendScore <= -trend_min_score signal_long = trend_ok_long AND close_pos >= 0.65 AND body_ratio >= 0.45 AND close > open AND pullback_legs >= 2 signal_short = trend_ok_short AND close_pos <= 0.35 AND body_ratio >= 0.45 AND close < open AND pullback_legs >= 2 entry_long = signal_high + tick entry_short = signal_low - tick stop_long = min(low[-pullback_lookback:]) - tick stop_short = max(high[-pullback_lookback:]) + tick target_long = entry_long + risk_reward * (entry_long - stop_long) target_short = entry_short - risk_reward * (stop_short - entry_short)
H2/L2 二次顺势入场

H2/L2 不是预测反转，而是趋势内第二次回撤失败后的继续跟随。多头只在信号K高点上方挂单，空头只在信号K低点下方挂单；虚线挂单、虚线止损、虚线目标都来自信号K已完成后的数值。

risk_long = entry_long - stop_long risk_short = stop_short - entry_short valid_risk = risk > 0 AND risk_pct <= max_structure_risk_pct filled = next_high >= entry_long OR next_low <= entry_short

开仓决策流水线

1. 背景条件先判断同级别是否已经是可交易趋势，只允许顺势方向进入下一步。TrendScore >= 1.0 OR TrendScore <= -1.0
2. 信号K条件信号K必须有方向、实体、收盘位置和二次顺势结构，不把第一下反转当开仓。close_pos >= 0.65 body_ratio >= 0.45 pullback_legs >= 2
3. 订单条件信号K收完后挂 stop entry，下一根及以后穿越信号K极值才成交。entry_long = high_signal + tick entry_short = low_signal - tick
4. 风控条件结构止损、追价距离和 2R 目标同时有效才开仓，风险不合格直接拒单。risk = abs(entry - stop) target = entry +/- 2 * risk
背景因子合并趋势背景：斜率、结构、均线。三项共同决定只做多、只做空，还是只观察；其中任一项明显背离，信号K不直接开仓。
信号K因子合并信号质量：收盘位置、实体、H/L 腿。收盘位置决定方向确认，实体决定推进强度，H/L 腿决定是否属于二次顺势。
执行风控合并执行边界：挂单、止损、目标。入场价、结构止损和 2R 目标在信号K收完后一次性确定，后续只判断触发与风险是否合格。

因子核对表

slope_z：趋势斜率
计算公式
最近 20 根收盘价做线性回归，斜率再做 z-score 标准化。
默认判定
大于 0 贡献多头分，小于 0 贡献空头分；绝对值越大，背景越强。
盘面读法
连续 K 线重心抬高或降低，均线有明确倾斜。
失败分支
斜率接近 0 时进入观察，不用单根大 K 直接开仓。
structure_score：高低点结构
计算公式
用摆动点统计 HH/HL/LH/LL，structure_score = sign(HH + HL - LH - LL)。
默认判定
HH/HL 占优给多头分，LH/LL 占优给空头分。
盘面读法
多头看回撤低点抬高，空头看反弹高点降低。
失败分支
高低点交错且重叠 K 增多，按中部震荡处理。
ma_alignment：均线方向
计算公式
ma_alignment = sign(EMA20 - EMA50)。
默认判定
EMA20 在 EMA50 上方偏多，下方偏空。
盘面读法
均线顺序和 K 线重心一致时，顺势信号可信度更高。
失败分支
均线缠绕时不把 H/L 信号当趋势延续。
close_pos 与 body_ratio：信号K质量
计算公式
close_pos = (close - low) / max(high - low, eps)；body_ratio = abs(close - open) / max(high - low, eps)。
默认判定
多头 close_pos 不低于 0.65，空头不高于 0.35；实体占比不低于 0.45。
盘面读法
强信号K应收在方向端，实体足够，影线不能主导整根 K。
失败分支
收盘回到中部或实体太小，只能作为观察信号。
follow_through：短线推进
计算公式
多头 close 高于最近 3 根短线高点，空头 close 低于最近 3 根短线低点。
默认判定
有推进才允许生成挂单，推进不足降低信号等级。
盘面读法
信号K必须把回撤后的方向重新推出来。
失败分支
新高/新低幅度太小且影线长，优先归为末端衰竭。
pullback_legs：H/L 腿
计算公式
统计最近回撤中逆趋势摆动次数，默认至少 2 段形成 H2/L2。
默认判定
H2/L2 比 H1/L1 更稳健，减少第一次反转误判。
盘面读法
趋势中回撤两次失败后，信号K重新顺势收强或收弱。
失败分支
只有第一段反向失败时，通常只观察，不急于开仓。

因子计算方式

slope_zslope_z = zscore(linreg_slope(close, 20))衡量最近 20 根 K 的斜率是否显著；斜率越陡，趋势背景越强。
structure_scorestructure_score = sign(HH + HL - LH - LL)用摆动高低点判断结构，多头看 HH/HL，空头看 LH/LL。
ma_alignmentma_alignment = sign(EMA20 - EMA50)短均线在长均线上方加多头分，在下方加空头分。
close_strengthclose_pos = (close - low) / max(high - low, eps) body_ratio = abs(close - open) / max(high - low, eps)收盘位置和实体占比决定信号K有没有推进质量。
follow_throughfollow_through = sign(close - highest(close[-3:-1]))多头看是否推进到短线新高，空头对称看是否打出短线新低。
pullback_legspullback_legs = count_countertrend_swings(last_pullback)默认更偏好两段回撤后的 H2/L2，减少第一下追高或追空。

可能性展示：早期顺势 / 中段回撤 / 末端衰竭

早期顺势

趋势刚形成，H1/H2 可以更积极；只要风险距离合理，突破信号K高点后顺势跟随。

中段回撤

核心看 H2/L2：回撤不能破坏结构，信号K必须重新收强；未穿越挂单价就不追。

末端衰竭

趋势过久、远离均线、影线失败或推进不足时，优先过滤末端假突破，不把高位强 K 当新趋势起点。

可能性矩阵

正常开仓

背景、信号K、挂单、风险全部通过，后续 K 穿越挂单价，生成实际开仓。

有效未触发

信号K成立，但后续价格没有突破信号K极值；记录为 no_fill，不追市补单。

风险拒单

挂单被触发，但结构止损太远或跳空追价太多；归因到撮合拒单。

过滤观察

趋势持续过久、推进不足或影线失败，进入过滤拒单或观察，不生成开仓。

触发条件拆解

趋势策略的触发条件不是单根强 K，而是“背景条件 + 信号K条件 + 订单条件 + 风控条件”同时成立。任何一层不满足，都只进入观察或拒单分支。

背景条件20 根 K 趋势评分进入 bull 或 bear 状态，均线和结构方向一致；多头只找 H1/H2，空头只找 L1/L2。
信号K条件信号K必须已经收完，方向、实体占比、收盘位置和回撤腿数达标；H2/L2 代表第二次顺势入场，优先级高于第一次追突破。
订单条件多头在信号K高点上方 1 tick 挂单，空头在信号K低点下方 1 tick 挂单；只有后续 K 穿越挂单价才进入撮合。
风控条件结构止损必须在入场价反方向，实际成交后重算结构风险、追价距离和 2R 目标；风险超限就是拒单，不临时改止损。
1. 背景先行

多头只在 TrendScore >= 1.0 且状态为 bull 时找 H1/H2；空头只在 TrendScore <= -1.0 且状态为 bear 时找 L1/L2。趋势不成立时，强 K 只算观察，不直接开仓。

2. 信号K必须有质量

多头要求收盘靠近高位、实体占比足够且阳线；空头要求收盘靠近低位、实体占比足够且阴线。影线过长、实体过小、收盘回到中部，按失败推进处理。

3. 挂单不是追市

多头在信号K高点上方 1 tick 挂 stop entry；空头在信号K低点下方 1 tick 挂 stop entry。下一根 K 没有穿越挂单价，订单就是 no_fill，不补做市价单。

4. 结构止损先确定

多头止损取近 5 根回撤低点下方；空头止损取近 5 根反弹高点上方。若止损距离过大，应该用最大结构风险过滤，而不是临时收窄止损。

信号分类

H1 / L1第一段回撤或反弹后的顺势信号，适合强趋势、浅回撤、均线继续同向的盘面；如果趋势已经走了很久，H1 更容易是末端追单。
H2 / L2第二段回撤或反弹后的顺势信号，是默认更偏好的信号。它给市场一次失败反向尝试，进场位置通常比第一下追突破更可控。
强趋势连续信号pullback_legs 不足但趋势评分很高时，系统可能给 H1/L1。交易员应检查是否已经远离均线和通道中轴，避免连续追高或追空。
反向强 K趋势中第一次反向强 K 不直接反手。它更常见的用途是减仓、观察或等待第二次确认。

触发矩阵

有效信号趋势评分、方向、收盘位置、实体占比和 H/L 腿数都满足；系统只生成挂单，不在信号K内部提前成交。
有效但未触发信号成立，但后一根及后续 K 没有穿越信号K高点或低点。复盘时记为未触发，不改成市价追单。
背景不满足趋势评分没有进入 bull/bear，或结构已经变成宽幅横盘。此时没有开仓信号，即使单根 K 很强，也只能归为观察。
信号成立未触发背景和信号K都达标，但后续 K 没有穿越挂单价。它说明市场没有给跟随，不属于漏单，也不应补做市价单。
触发后风险不合格价格穿越挂单价后，若跳空、追价或结构止损距离让实际风险超限，归为触发后拒单；信号仍保留，执行层单独归因。
过滤拒单交易方向不允许、大周期方向不一致、信号K无流动性、末端假突破风险或同级别中部震荡，不进入撮合层。
撮合拒单挂单被触发后，实际结构风险过大、追价过远、目标价无效或订单字段异常，由撮合层拒绝。
持仓冲突单策略满仓进出，一笔未平仓前又出现同向或反向信号，记为已有持仓未平仓。

信号生命周期

无信号趋势评分、回撤腿数或信号K质量不达标。交易员应继续等待，不要把强 K 倒推成趋势。
观察信号方向或强度有迹象，但处在第一次反转、同级别中部或末端衰竭区。观察信号可以用于减仓或复盘，不生成开仓。
有效未触发信号K成立，但后续 K 没有突破信号K极值。它说明市场没有给出跟随，不应改成主观追单。
触发后拒单挂单价被穿越，但真实成交价导致追价过远、结构风险过大或目标价无效。拒单属于执行质量问题，不应回头修改信号。
退出完成成交后只按结构止损、固定目标、回撤止盈、最大持有 K 数或样本结束退出；退出原因要回写到交易路径。

或然情况分层

识别阶段无信号、观察信号、有效信号。观察信号包括第一次反转、趋势中部弱 K、末端衰竭 K，不参与开仓统计。
触发阶段有效但未触发、正常触发、跳空触发。未触发说明市场没有给出跟随，跳空触发要按真实开盘价重算风险。
过滤阶段方向禁用、大周期方向不一致、信号K无流动性、末端假突破、同级别中部不交易，都会写入策略过滤日志。
撮合阶段追价超限、结构止损风险超限、目标价无效、订单字段异常属于执行质量问题，和趋势 detector 的信号质量分开复盘。
持仓/退出阶段已有持仓冲突、结构止损、固定目标、最大盈利回撤止盈、均线回撤止盈、最大持有 K 数和样本结束分别归因。

或然情况

触发后同 K 触及止损和目标默认按保守规则处理，先算止损。盘面复盘时不要把无法确认先后顺序的 K 线当成确定盈利。
跳空越过挂单价按实际可成交开盘价入场，真实风险和盈亏比会重算；若风险超限，应由撮合层拒单。
信号后不触发不是错过交易，而是信号没有得到下一根 K 的确认。趋势策略重视“信号K + 触发K”的组合。
同级别中部震荡单趋势策略不会自动调用区间 detector；手工盘面若看到重叠 K 多、均线走平、上下边界清楚，应降低趋势信号权重。
末端假突破趋势持续很久、价格远离中轴、突破推进不足、上影线或下影线明显时，应开启末端假突破过滤或直接放弃追单。

策略空间

策略空间不是无限可做，而是先限定可交易盘面，再限定参数、过滤、执行、退出和统计复盘边界。

可交易空间清晰单边趋势、回撤不深、均线和结构同向、信号K有实体推进。核心收益来自顺势延续，不来自预测反转。
盘面阶段早期顺势可以接受 H1/H2 或 L1/L2；中段回撤优先看二次入场质量；末端衰竭要检查远离均线、影线失败和突破推进不足。
参数空间趋势窗口、趋势阈值、收盘位置、实体占比、回撤腿数、最大结构风险、追价上限、盈亏比、最大持有 K 数共同决定信号密度和风险距离。
过滤空间交易方向、多周期趋势一致、末端假突破、同级别中部不交易、信号K流动性，都是开仓前过滤，不改变 detector 本身。
订单空间订单只来自有效信号K：多头高点上方挂单，空头低点下方挂单。订单状态拆成未触发、正常触发、跳空触发和触发后拒单。
风险空间风险先由结构止损距离定义，再由最大结构风险、追价上限、目标价方向和同 K 止盈止损规则约束。风险不合格就拒单，不临时改价。
持仓空间单策略只做满仓进出；一笔交易未平仓前，后续同向或反向信号都记录为持仓冲突，组合仓位分配不在本策略内处理。
执行空间信号K只产生挂单价格和结构止损；是否成交由后续 K 是否穿越挂单价决定，跳空成交按真实入场价重算。
退出空间结构止损、2R 固定目标、最大持有 K 数、盈利通道回撤止盈可组合使用；退出模块独立于信号识别。
统计空间逐笔交易、订单决策、策略过滤、信号形态绩效、交易路径分布和回撤区间分别记录，让有效信号、未触发和拒单都能复盘。
失效空间宽震荡、假突破密集、趋势末端、第一次反转、消息跳空后的无序波动，都会削弱 H/L 信号质量。
执行边界单策略只负责产生订单和满仓进出；组合策略才处理策略优先级、资金上限、行业约束和多持仓冲突。
