Al Brooks Always-In 趋势策略算法草图

把 Brooks 的 always-in、市场周期、信号K/入场K、H2/L2、突破挂单和失败突破，重设计为可落地到 5m / 15m / 30m / 60m 的状态机策略；不做打板，不做同级别中部，不做第一次反转。

来源：本地 albrooks_combined_pages.html 的结构化总结。这里是算法化设计，不摘录原文。所有文字可直接点击修改。

总纲：不是识别单一形态，而是维护“必须持仓时的方向”

1. Always-In 先行 每根 K 后更新 AI_LONG / AI_SHORT / RANGE / WATCH。没有足够反证前，不轻易从趋势切到反转。
2. 背景大于信号 同一根信号K，在强趋势里是回撤入场，在区间中可能只是中部噪声。
3. 入场用 stop order 突破只在信号K上方/下方挂单进场；不等回测，不靠主观追价。
4. 涨停开盘过滤 日K开盘等于该板块涨停价的样本直接过滤，避免打板和一字板影响回测。
5. 实际风险过滤 先确定结构止损，再看 entry 到 stop 的风险是否可接受；错过好价位就跳过。
AI
LONG
偏多持仓状态 优先找 H1/H2、信号K高点上方挂单、小回撤趋势延续；空头信号只当减仓或观察。
AI
SHORT
偏空持仓状态 优先找 L1/L2、信号K低点下方挂单、小反弹趋势延续；多头信号只当离场或观察。
RANGE
交易区间状态 同级别中部不交易，只在上下沿做失败突破；强突破和跟随出现后才切趋势。
WATCH
过渡与反转观察 第一次反转只观察或离场；旧趋势破线、回测旧极端失败、第二次强反向信号齐备后，才允许切换 always-in 方向。

算法架构

模式识别必须解耦：趋势、通道、区间、反转分别是独立 detector。单一策略回测只加载一个 detector 和对应 trigger，不牵涉其他模块；组合策略回测再由 portfolio 层统一处理信号冲突、资金占用和仓位分配。

原则：detector 只识别模式并输出标准事件；strategy 只消费事件；portfolio 才处理多策略组合和仓位。

L0 TDX K线 5m / 15m / 30m / 60m OHLCV，沿用现有 TDX-only 数据层。
L1 Bar Features 实体、上下影、收盘位置、重叠率、连续同向收盘、跟随K。
L2 独立 Detectors trend / channel / range / reversal 分开计算，各自输出事件，不互相调用。
L3 Strategy Adapter 单策略只绑定一个 detector，例如 TrendH2Strategy 只消费 trend events。
L4 Trigger + Risk 信号K挂单、信号K流动性过滤、结构止损、实际风险过滤和零流动性拒单，仍属于单策略内部。
L5 Backtest Mode single_strategy 只测一个策略；portfolio_strategy 才组合多策略与仓位。
TrendDetector 只识别趋势状态、H1/H2、L1/L2、小回撤趋势。输出 trend_state、pullback_legs、signal_bar，不读取区间或反转模块。
ChannelDetector 只识别回归通道、摆动点通道、通道位置、通道破坏或过冲。输出 upper/lower/channel_pos/slope，不决定是否交易。
RangeDetector 只识别区间上下沿、区间中部、失败突破。输出 range_high/range_low/range_pos/failed_break，不调用趋势策略。
ReversalDetector 只识别第一次反转观察、第二次反转确认、CHoCH、旧极端测试失败。默认不产生第一笔反手交易。
单一策略回测 输入：一个 detector 的事件流。输出：该策略自己的订单、成交、止损止盈、收益曲线。不得读取其他 detector 的状态，不做仓位竞争，不做资金再分配。
组合策略回测 输入：多个单策略信号。由 portfolio 层处理信号优先级、互斥规则、最大持仓数、单票/行业暴露、资金占用和动态仓位分配。

核心规则定义

先把 Brooks 语言转成可计算字段。这里的阈值是第一版参数，后续回测再调；各 detector 独立输出事件，单策略只消费自己的事件。

重点：不要让单根漂亮 K 线越过背景过滤。

0. 解耦回测边界

单策略回测不做跨模块推理；组合策略回测另设 portfolio 层。

# 每个 detector 只负责一种模式识别，输出标准事件，不直接下单。 DetectorEvent = { detector_name, # trend / channel / range / reversal symbol, timeframe, bar_time, event_type, # 例如 H2_SETUP、CHANNEL_BREAK、FAILED_BREAKDOWN direction, # long / short / neutral signal_bar_id, confidence, metadata # 仅保存本 detector 内部字段 } # 单策略回测：只绑定一个 detector，禁止读取其他 detector 状态。 SingleStrategyBacktest(detector_name): events = run_detector(detector_name, bars) orders = strategy.consume(events) strategy_filter_decisions keep accepted / signal_bar_no_liquidity / invalid_signal_order wrapped_strategy_filter_decisions = base_filter_decisions + higher_timeframe_filter_decisions trades = execution.simulate(orders) trades keep order_id / event_id / signal_bar_index / planned_entry / stop / target / r_multiple / mae / mfe / metadata order_decisions keep accepted / no_fill / no_liquidity / already_open / no_bars trades sorted by actual entry_time, not stock_code return single_strategy_metrics(trades) # 组合策略回测：只在 portfolio 层合并多个单策略结果。 PortfolioBacktest(strategies): all_signals = collect(strategy.signals for strategy in strategies) candidate_trades = execution.simulate(all_signals) candidate_trades sorted by actual entry_time, priority, symbol, order_id each candidate risk_fraction = risk_per_share / actual_entry_price trades, order_decisions = portfolio_allocator.allocate(candidate_trades) order_decisions keep status / reason / capital_fraction / risk_fraction rejected reasons include no_fill / no_liquidity / no_bars / max_open_positions / same_symbol_overlap / no_capital equity_curve = mark_to_market(trades, bars) return { trade_metrics: win_rate / profit_factor / expectancy, equity_metrics: total_return / max_drawdown / annualized_return / calmar_ratio, exposure_metrics: avg_gross_exposure / max_gross_exposure / avg_open_positions }

0A. 数据覆盖审计

回测前必须知道本地分钟 K 是否完整，也要知道哪些 K 线没有真实成交。覆盖度只统计已观测交易日内的 5m/15m/30m/60m 交易时段，不把未观测整日猜成交易日。

# A股分钟K预期时间轴，按上午和下午两个交易时段生成。 SessionTimes(tf, day) = range(09:30 + tf, 11:30, step=tf) UNION range(13:00 + tf, 15:00, step=tf) ExpectedRows = count(SessionTimes(tf, observed_session_days)) ZeroLiquidityRows = count(Volume == 0 OR Amount == 0) TradableObservedRows = count(unique(bar_time in SessionTimes AND Volume > 0 AND Amount > 0)) MissingRows = ExpectedRows - TradableObservedRows CoverageRatio = TradableObservedRows / max(ExpectedRows, 1) MaxMissingGapMinutes = max_consecutive_missing_bars * tf_minutes # 严格质量门禁默认对缺文件、空窗口、缺字段、重复K、非法OHLC失败。 # 零流动性K不计入可交易覆盖率，进入审计提示，回测数据包传给 detector 前先剔除。 # 裸策略/撮合入口仍由 strategy/execution 层二次拒绝。 # 如果设置 min_coverage_ratio，则覆盖率也进入硬门禁。 If min_coverage_ratio is set AND ExpectedRows > 0 AND CoverageRatio < min_coverage_ratio: DataGate = FAIL

1. K 线质量

判断趋势K、信号K、入场K的基础。收盘位置和实体占比优先于形态名字。

# K线质量用于判断趋势K、信号K、入场K，先量化单根K线本身。 Range = High - Low # 全振幅，衡量这一根K的波动范围。 Body = abs(Close - Open) # 实体长度，衡量多空一方的控制力度。 ClosePos = (Close - Low) / max(Range, eps) # 收盘位置，越接近1越偏多，越接近0越偏空。 BodyRatio = Body / max(Range, eps) # 实体占比，过滤十字星和犹豫K。 OverlapRatio = overlap(CurrentRange, PrevRange) / Range # 与上一根K的重叠程度，重叠越高越偏区间。 # 强牛K：上涨、收在高位、实体足够大。 StrongBullBar = Close > Open AND ClosePos >= 0.65 AND BodyRatio >= 0.45 # 强熊K：下跌、收在低位、实体足够大。 StrongBearBar = Close < Open AND ClosePos <= 0.35 AND BodyRatio >= 0.45

2. 市场周期评分

把“趋势、通道、区间、过渡”变成互斥倾向，不直接用一个通道突破替代全局状态。

# 市场周期评分用于先判背景，再决定是否允许开仓。 # BullScore 越高，说明市场越接近多头趋势。 BullScore = HigherHighCount(N) # 近期更高高点数量，代表上攻结构。 + HigherLowCount(N) # 近期更高低点数量，代表回撤被承接。 + CloseAboveEMA20Count(N) # 收在均线上方的K数，代表多头持续性。 + StrongBullBarCount(N) # 强牛K数量，代表主动买盘。 + FollowThroughUpCount(N) # 突破后的跟随K数量，代表突破质量。 - DeepPullbackCount(N) # 深回撤扣分，避免把宽震荡当趋势。 # RangeScore 越高，说明市场越接近交易区间。 RangeScore = OverlapBarCount(N) # K线重叠数量，重叠多说明没有单边效率。 + TailBarCount(N) # 长影线数量，说明上下都容易被打回。 + FailedBreakoutCount(N) # 失败突破数量，说明区间边界有效。 + EMAFlatness(N) # 均线走平程度，趋势斜率不足。 - FollowThroughCount(N) # 有效跟随K越多，越不像区间。

3. Always-In 切换

方向切换必须有强证据。趋势中第一次破线默认先看成回撤或过渡，不立刻反手。

# Always-In 状态每根K更新一次，交易信号必须服从状态。 # 多头评分明显强于空头，且不是区间，切到 AI_LONG。 If BullScore - BearScore >= trend_threshold AND RangeScore < range_threshold: State = AI_LONG # 空头评分明显强于多头，且不是区间，切到 AI_SHORT。 If BearScore - BullScore >= trend_threshold AND RangeScore < range_threshold: State = AI_SHORT # 区间评分高，且多空评分差距不大，切到 RANGE。 If RangeScore >= range_threshold AND abs(BullScore - BearScore) < neutral_band: State = RANGE # 从空头切多头必须有破下降通道、回测失败和强牛信号。 FlipToLong requires: PriorState == AI_SHORT AND BearChannelBroken AND TestOfOldLowFails AND StrongBullSignal

4. Setup 优先级

同一根 K 可能触发多个结构，按背景优先级选择，避免重复开仓。

# 同一根K可能满足多个条件，必须按状态优先级去重。 # AI_LONG：只找顺势买点，空头信号不反手。 AI_LONG priority: 1. SmallPullbackTrendContinuation # 小回撤趋势延续，优先级最高。 2. H2 after two-legged pullback # 两段回撤后的第二次多头入场。 3. SignalBarStopEntry above signal high# 信号K高点上方挂单。 4. FailedBreakdown at range low # 区间下沿假跌破后的多头机会。 # RANGE：同级别中部禁做，只处理区间边缘。 RANGE priority: 1. FailedBreakout at range high/low # 上沿/下沿失败突破。 2. BreakoutMode wait only # 突破模式只等待，不提前下注。 3. No trade in same-timeframe middle 50%# 当前级别区间中部不开仓。 # WATCH：反转观察，不做第一笔反手。 WATCH priority: 1. First reversal signal = exit/watch only # 第一次反转只观察或离场。 2. Second reversal signal after break + failed test# 第二次确认后才允许新方向挂单。 3. Otherwise no fresh reverse entry # 证据不足不反手。

数学化定义：结构、通道、趋势

所有主观描述必须落到公式字段。结构用分形摆动点和高低点序列，通道用滚动回归或摆动点趋势线，趋势用加权评分和状态转移。

实现时这些定义应变成独立模块：bar_features、market_structure、trend_channel、always_in_state、setup_detector。

1. 基础 K 线变量

所有结构判断先从 OHLCV 派生字段开始，不直接用形态名称。

# 第 i 根K线的基础字段，后续所有结构、通道、趋势判断都从这里派生。 K_i = (O_i, H_i, L_i, C_i, V_i) # 开、高、低、收、成交量。 Range_i = H_i - L_i # 单根K线全振幅。 Body_i = abs(C_i - O_i) # 实体长度。 ClosePos_i = (C_i - L_i) / max(Range_i, eps) # 收盘在本根K线中的相对位置。 BodyRatio_i = Body_i / max(Range_i, eps) # 实体占全振幅比例。 UpperTail_i = H_i - max(O_i, C_i) # 上影线长度。 LowerTail_i = min(O_i, C_i) - L_i # 下影线长度。 ATR_i = mean(TrueRange, N_atr) # 平均真实波幅，用于统一不同价格股票的波动尺度。 Ret_i = log(C_i / C_{i-1}) # 对数收益率，用于趋势斜率标准化。
1A. 日K涨停开盘过滤

不考虑打板策略。若日K开盘价等于该板块涨停价，则该标的当日所有分钟级信号从回测样本中剔除。

# 根据股票代码或交易所板块确定涨停幅度，用于过滤打板/一字板样本。 BoardLimitPct(symbol, board) = 0.10 if board in [MAIN_SH, MAIN_SZ] # 主板普通股票按10%。 0.20 if symbol startswith 688 or board == STAR # 科创板按20%。 0.20 if symbol startswith 300/301 or board == CHINEXT # 创业板按20%。 0.30 if board == BJ or symbol startswith 4/8/920 # 北交所按30%。 PrevClose_D = DailyClose_{D-1} # 前一交易日收盘价。 LimitUpOpen_D = round_to_tick(PrevClose_D * (1 + BoardLimitPct), 0.01) # 理论涨停开盘价，按0.01元最小价位取整。 LimitOpenFiltered_D = abs(DailyOpen_D - LimitUpOpen_D) <= limit_open_tick_tolerance OR DailyOpen_D / PrevClose_D - 1 >= BoardLimitPct - limit_open_pct_tolerance # 价格精确等于或接近涨停开盘，都视为需过滤。 If LimitOpenFiltered_D: exclude all intraday bars of symbol on D # 剔除该股票当日所有分钟K。 exclude all entries whose signal_date == D # 同时剔除当日所有入场信号。
2. 摆动点与市场结构

结构不是肉眼画线，先定义 pivot high / pivot low，再计算 HH、HL、LH、LL。

# m 是摆动点确认窗口；必须等右侧 m 根K完成后，pivot 才算确认。 PivotHigh_i = H_i == max(H_{i-m ... i+m}) # 当前高点是左右窗口最高点。 AND H_i > max(H_{i-m ... i-1}) # 当前高点必须高于左侧历史高点。 PivotLow_i = L_i == min(L_{i-m ... i+m}) # 当前低点是左右窗口最低点。 AND L_i < min(L_{i-m ... i-1}) # 当前低点必须低于左侧历史低点。 # 用确认后的摆动点判断结构：更高高点、更高低点、更低高点、更低低点。 HH_i = PivotHigh_i AND PH_i > last(PH) LH_i = PivotHigh_i AND PH_i <= last(PH) HL_i = PivotLow_i AND PL_i > last(PL) LL_i = PivotLow_i AND PL_i <= last(PL) # 结构分数为正偏多，为负偏空。 StructureScore_N = count(HH, N) + count(HL, N) - count(LH, N) - count(LL, N)
3. 结构突破 BOS / CHoCH

突破结构必须基于已确认摆动点，避免当前 K 自己制造参照物。

# 结构突破必须参考已经确认的摆动点，不能用当前K临时制造参照。 LastSwingHigh_i = last confirmed PivotHigh before i # 最近一个已确认摆动高点。 LastSwingLow_i = last confirmed PivotLow before i # 最近一个已确认摆动低点。 BOS_Up_i = C_i > LastSwingHigh_i * (1 + bos_buffer) # 收盘突破前摆动高点。 AND ClosePos_i >= strong_close_pos # 且收盘质量足够强。 BOS_Down_i = C_i < LastSwingLow_i * (1 - bos_buffer) # 收盘跌破前摆动低点。 AND ClosePos_i <= 1 - strong_close_pos # 且收盘质量足够弱。 CHoCH_Up_i = prior State == AI_SHORT # 原状态为空头。 AND BOS_Up_i # 出现向上结构突破。 AND BearChannelBroken_i # 同时破下降通道，才进入反转观察。
4. 回归趋势通道

通道第一版用 log(close) 滚动回归，残差和 ATR 共同确定宽度。

# 回归通道用于客观描述趋势斜率和通道位置。 For window W = [i-N+1 ... i]: # 取最近 N 根K作为通道窗口。 x_t = 0 ... N-1 # 时间序列编号。 y_t = log(C_t) # 用收盘价对数，减少价格尺度影响。 beta_i = cov(x, y) / var(x) # 回归斜率，代表趋势速度。 alpha_i = mean(y) - beta_i * mean(x)# 回归截距。 MidLog_t = alpha_i + beta_i * x_t # 通道中线的对数价格。 Resid_t = y_t - MidLog_t # 价格偏离中线的残差。 Sigma_i = 1.4826 * median(abs(Resid - median(Resid))) # 稳健残差波动，降低异常K影响。 Upper_i = exp(MidLog_i + k_sigma * Sigma_i) # 通道上轨。 Lower_i = exp(MidLog_i - k_sigma * Sigma_i) # 通道下轨。 ChannelPos_i = (C_i - Lower_i) / max(Upper_i - Lower_i, eps) # 当前价格在通道中的位置，0靠下轨，1靠上轨。 SlopeZ_i = beta_i / max(std(Ret, N), eps) # 标准化斜率，用于跨股票比较趋势强弱。
5. 摆动点趋势通道

第二版可用摆动点画人工风格通道，适合 Brooks 的趋势线/通道线语言。

# 摆动点通道更接近人工画线：用两个确认低点画上升支撑线。 BullSupportLine: use last two confirmed PivotLow: (t1, PL_1), (t2, PL_2) # 最近两个已确认摆动低点。 support(t) = PL_1 + (PL_2 - PL_1) * (t - t1) / (t2 - t1) # 由两个低点外推得到支撑线。 ChannelWidth = max(H_t - support(t), t in W) # 通道宽度取窗口内高点到支撑线的最大距离。 UpperSwing(t) = support(t) + ChannelWidth # 上轨。 LowerSwing(t) = support(t) # 下轨，即支撑线。 BullChannelBreak_i = C_i < LowerSwing(i) * (1 - channel_break_buffer) # 收盘跌破上升通道下轨，趋势受损。 OR C_i > UpperSwing(i) * (1 + channel_overshoot_buffer) # 收盘过冲上轨，可能进入高潮或获利区。
6. 趋势评分

趋势由斜率、结构、均线位置、收盘强度、跟随K、回撤深度共同决定。

# 趋势评分综合斜率、结构、均线、收盘质量和跟随K。 MAAlign_i = 1 if C_i > EMA20_i > EMA50_i # 多头均线排列。 -1 if C_i < EMA20_i < EMA50_i # 空头均线排列。 0 otherwise # 均线缠绕，不加方向分。 FollowUp_N = count(C_t > H_{t-1}, N) # 向上跟随K数量。 FollowDown_N = count(C_t < L_{t-1}, N) # 向下跟随K数量。 DeepPullback_N = count(ChannelPos_t < 0.25, N) # 多头趋势中的深回撤次数。 DeepBounce_N = count(ChannelPos_t > 0.75, N) # 空头趋势中的深反弹次数。 TrendScore_i = w_slope * SlopeZ_i # 趋势斜率贡献。 + w_struct * StructureScore_N # 高低点结构贡献。 + w_ma * MAAlign_i # 均线排列贡献。 + w_close * mean(2 * ClosePos_t - 1, N) # 收盘质量贡献。 + w_follow * (FollowUp_N - FollowDown_N) # 跟随K方向贡献。 - w_pullback * DeepPullback_N # 深回撤扣分。
7. 交易区间评分

区间不是“没趋势”，而是重叠多、突破失败多、均线斜率弱、上下沿清楚。

# 区间评分用于识别同级别震荡，避免在中部开仓。 Overlap_i = intersection([L_i,H_i], [L_{i-1},H_{i-1}]) / max(Range_i, eps) # 与上一根K重叠比例。 TailRatio_i = (UpperTail_i + LowerTail_i) / max(Range_i, eps) # 上下影线占比，影线多说明突破易失败。 EMAFlat_i = abs(EMA20_i - EMA20_{i-N}) / max(ATR_i, eps) # 均线斜率越小，越偏区间。 RangeHigh_i = max(PivotHigh, N_range) # 区间上沿，取窗口内摆动高点最大值。 RangeLow_i = min(PivotLow, N_range) # 区间下沿，取窗口内摆动低点最小值。 RangeWidthATR_i = (RangeHigh_i - RangeLow_i) / max(ATR_i, eps) # 区间宽度用ATR标准化。 RangeScore_i = w_overlap * mean(Overlap, N) # 重叠越多，区间分越高。 + w_tail * mean(TailRatio, N) # 影线越多，区间分越高。 + w_fail * FailedBreakoutCount_N # 失败突破越多，边界越可靠。 - w_slope * abs(SlopeZ_i) # 斜率越强，越不像区间。 - w_width * max(RangeWidthATR_i - max_range_atr, 0) # 区间过宽时扣分，避免混成宽通道。 RangePos_i = (C_i - RangeLow_i) / max(RangeHigh_i - RangeLow_i, eps) # 当前价格在区间里的相对位置。 NoTradeSameTfMiddle_i = State_i == RANGE # 只有在同级别被判为区间时生效。 AND RangePos_i >= 0.25 AND RangePos_i <= 0.75 # 25%-75%定义为中部禁做区。
8. Always-In 状态转移

状态是策略核心，交易信号必须服从状态，不允许形态单独开仓。

# Always-In 是状态机，不是单根K形态判断。 # 趋势分达到阈值且不是区间，才允许切趋势状态。 If TrendScore_i >= trend_on AND RangeScore_i < range_on: State_i = AI_LONG # 多头趋势状态，只优先找多头setup。 If TrendScore_i <= -trend_on AND RangeScore_i < range_on: State_i = AI_SHORT # 空头趋势状态，只优先找空头setup。 If RangeScore_i >= range_on AND abs(TrendScore_i) < trend_neutral: State_i = RANGE # 区间状态，同级别中部不开仓。 If prior AI_SHORT and CHoCH_Up_i: State_i = WATCH # 空转多的第一次结构变化，只进入观察。 FirstReversalSignal_i = State_{i-1} in [AI_LONG, AI_SHORT] # 原来处在明确趋势中。 AND opposite CHoCH appears # 出现反向结构变化。 Action = exit_or_watch_only # 第一次反转只离场或观察，不反手。 SecondReversalSignal_i = prior FirstReversalSignal exists within N bars # N根内出现过第一次反转。 AND TestOfOldExtremeFails_i # 回测旧高/旧低失败。 AND StrongOppositeSignal_i # 出现强反向信号K。 State flip from WATCH to AI_LONG/AI_SHORT requires: SecondReversalSignal_i # 必须是第二次确认。 AND TestOfOldExtremeFails_i # 旧趋势极端位置测试失败。 AND stop entry is triggered beyond SignalBar # 必须上破/下破信号K触发挂单。 AND optional FollowThrough_{i+1} # 成交后可用下一根跟随K决定持有强度。
9. H2 / L2 数学定义

H2/L2 不是机械数两根 K，而是趋势背景下两段回撤后的第二次顺势触发。

# H2/L2 用来交易趋势中的两段回撤，不是逆势反转。 BullPullback_i = State_i == AI_LONG # 只有多头 always-in 状态才找 H2。 AND C_i < recent swing high # 价格从最近高点回撤。 AND ChannelPos_i <= pullback_channel_pos # 回撤进入通道中下部，风险更合理。 DownLeg = sequence from local high to next PivotLow with lower close or lower low # 下跌腿以更低收盘或更低低点确认。 BullH2Setup_i = State_i == AI_LONG AND count(DownLeg since last swing high) >= 2 # 至少两段回撤，避免第一脚就追。 AND min(TrendScore during pullback) > -trend_neutral # 回撤过程中不能变成强空头。 AND StrongBullBar_i # 当前必须出现强牛信号K。 Entry_i = H_i + tick # 多头在信号K高点上方挂单。 Stop_i = min(L_i, PullbackLow_i) - tick # 止损放在信号K低点或回撤低点外侧。
10. 突破与信号K挂单

突破不等回测。只在信号K高点上方或低点下方挂 stop order；成交后再用跟随K决定持有或退出。

# 突破只做信号K挂单，不等待回测。 BreakoutUp_i = C_i > max(RangeHigh_{i-1}, Upper_{i-1}) * (1 + breakout_buffer) # 收盘突破前一根已知压力或通道上轨。 AND ClosePos_i >= strong_close_pos # 收在高位，说明突破质量较好。 AND BodyRatio_i >= min_body_ratio # 实体足够大，过滤假突破和十字K。 BullSignalBar_i = StrongBullBar_i # 当前是强牛信号K。 AND BreakoutUp_i # 同时完成突破。 AND NOT NoTradeSameTfMiddle_i # 不能位于同级别区间中部。 LongStopEntry_i = H_i + tick # 买入 stop 挂在信号K高点上方。 LongStop_i = min(L_i, PullbackLow_i, Lower_i) - tick # 结构止损取信号K低点、回撤低点、通道下轨更保守者。 EntryTriggered_{i+1} = H_{i+1} >= LongStopEntry_i # 下一根触及挂单价才成交。 FollowThroughUp_{i+1} = C_{i+1} > LongStopEntry_i # 成交K收在入场价上方。 OR C_{i+1} > H_i # 或直接突破信号K高点，说明有跟随。
11. 失败突破

区间策略只做极端。失败突破要求先突破区间，再收回区间，并出现反向信号。

# 失败突破只在区间边缘交易，中部禁做。 FailedBreakdown_i = State_{i-1} == RANGE # 前一根处在同级别区间状态。 AND L_i < RangeLow_{i-1} * (1 - false_break_buffer) # 盘中跌破区间下沿。 AND C_i > RangeLow_{i-1} # 收盘重新回到区间内。 AND ClosePos_i >= strong_close_pos # 收盘靠上，说明下破失败。 FailedBreakout_i = State_{i-1} == RANGE # 前一根处在同级别区间状态。 AND H_i > RangeHigh_{i-1} * (1 + false_break_buffer) # 盘中突破区间上沿。 AND C_i < RangeHigh_{i-1} # 收盘重新回到区间内。 AND ClosePos_i <= 1 - strong_close_pos # 收盘靠下，说明上破失败。 NoTradeSameTfMiddle = State_i == RANGE AND C_i between 25% and 75% of [RangeLow, RangeHigh] # 当前价格处在区间中部。 AND signal_tf == range_tf # 只约束同级别；高低级别需另行共振。
12. 风险、仓位、目标

所有 setup 最终必须过实际风险过滤，不能只因为形态成立就交易。

# 风控先于收益目标；风险不合格的setup直接跳过。 ActualRiskPct_i = abs(Entry_i - Stop_i) / Entry_i # 入场价到结构止损的实际风险比例。 ValidRisk_i = ActualRiskPct_i <= max_actual_risk_pct # 单笔实际风险不能超过参数上限。 AND Entry_i is not worse than SignalBar close by max_chase_pct # 挂单成交价不能离信号K收盘太远，避免追高。 SignalBarStopStrategy rejects setup if: max_actual_risk_pct is set AND ActualRiskPct_i > max_actual_risk_pct OR max_chase_pct is set AND ChasePct_i > max_chase_pct # 两个上限都是可选参数；设空则关闭对应过滤。 PositionSize_i = floor(AccountEquity * risk_budget_pct / abs(FilledEntry_i - Stop_i)) # 按真实成交后的风险预算反推出仓位，跳空成交不低估风险。 Target_1R = Entry_i + direction * abs(Entry_i - Stop_i) # 1R目标。 Target_2R = Entry_i + direction * 2 * abs(Entry_i - Stop_i) # 2R目标。 MeasuredMoveTarget = BreakoutLevel + direction * RangeHeight # 区间高度测算目标。 RiskPerShare_i = abs(FilledEntry_i - Stop_i) # 成交后真实每股风险。 RMultiple_i = NetReturnPct_i / 100 * FilledEntry_i / RiskPerShare_i # 每笔交易最终赚亏多少 R。 NetReturnPct_i = direction * (SlippedExit_i / SlippedEntry_i - 1) * 100 - 2 * fee_rate * 100 # 成本模型进入所有单策略、组合和参数遍历回测。 SlippedEntry_i = RawEntryFill_i * (1 + direction * slippage_bps / 10000) SlippedExit_i = RawExitFill_i * (1 - direction * slippage_bps / 10000) # 滑点按买入抬高、卖出压低处理；空头方向相反。 MAE_pct_i = min(0, direction * (AdversePrice_i / FilledEntry_i - 1) * 100) # 持仓期最大不利波动百分比。 MFE_pct_i = max(0, direction * (FavorablePrice_i / FilledEntry_i - 1) * 100) # 持仓期最大有利波动百分比。 MAE_R_i = MAE_pct_i / 100 * FilledEntry_i / RiskPerShare_i MFE_R_i = MFE_pct_i / 100 * FilledEntry_i / RiskPerShare_i # 把路径波动统一换算成 R，方便比较不同价格股票。

K线呈现：三类可交易场景

先只做这三类，足够支撑第一版回测：趋势回撤顺势、同级别区间极端失败突破、二次反转确认。其它 Brooks 形态先归入这三类，不散开。

图里的文字也是策略口径，你可以直接改。

A. AI_LONG：H2 / 信号K上方挂单

高周期 always-in 偏多；回撤两段不转为空头；出现强牛信号K后，只在信号K高点上方挂单进场，不等回测。

B. RANGE：区间极端失败突破

区间里多数突破失败；同级别中部不做，只在上下沿外侧失败后回到区间内交易。

C. WATCH：第一次反转不做

第一次反转只用于观察或离场；旧趋势破线、回测旧低/旧高失败后，等第二次强反向信号，再按信号K挂单切换 always-in。
